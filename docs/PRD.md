# Tokenomy PRD (제품 요구사항 정의)

- 최종 갱신: 2026-06-17 (v0.1.8 — 단가 커버리지 진단·단가 변경 자동 재계산 반영)
- 상태: 내부 PoC → 공개(MIT) 범용화 진행 중. 로컬 전용 메모(커밋 제외).
- 출처: 브레인스토밍 세션 종합 + [`specs/2026-06-12-tokenomy-public-generalization-design.md`](superpowers/specs/2026-06-12-tokenomy-public-generalization-design.md)
- 관련: [ARCHITECTURE.md](ARCHITECTURE.md) · [ROADMAP.md](ROADMAP.md)

## 한 줄 정의

AI 코딩 토큰 지출용 **로컬 가계부**. 내 머신의 Claude Code / Codex CLI 세션 로그를 파싱해
월 예산 대비 사용량·비용·효율을 보여주고, 종량제 사용자가 월 중에 예산을 초과하지 않도록 돕는다.

## 문제 / 배경

- 종량제(API 과금) AI 코딩 도구는 **쓰는 만큼 청구**되는데, 사용자는 월 중 자기 누적 지출과
  소진 속도를 직관적으로 알기 어렵다. 청구서를 받고 나서야 초과를 안다.
- 기존 도구(ccusage 등)는 사용량 합계는 보여주지만 **"내가 설정한 예산 대비 번다운"**,
  **프로젝트/세션 단위 귀속**, **캐시 효율 신호**를 한 화면에서 묶어주지 않는다.
- 로그를 외부로 보내는 건 프라이버시·보안상 부담 — **완전 로컬**이어야 한다.

## 타깃 사용자

1. **1차: 종량제(API-metered) 사용자** — Claude Code / Codex CLI를 개인 키로 쓰며 월 지출을
   추적·상한 관리하려는 개인 개발자.
2. **2차(덤): 구독 정액제(Pro/Max/Plus) 사용자** — 비용은 *공개 API 단가 환산 추정치*로만
   제공. 사용량·효율 추적 용도.

## Goals

1. 사용자가 입력한 **provider별 예산** 대비 번다운·소진 예상일을 보여준다 — Claude는 월간 한도,
   Codex는 주간 누적(월÷4 한도가 매주 월요일 충전·미사용분 월 내 이월). 예산 도입일(`budget_start`)을
   지정하면 그 달은 도입일부터 계산한다.
2. **프로젝트/세션 단위 비용 귀속** — 어디에 얼마를 썼는지, 세션 작업요약(aiTitle)과 함께 복기.
3. **캐시 효율 신호** — cache_read 비율·web_search 빈도 등으로 낭비 가능성을 코칭(단정 아닌 신호).
4. **완전 로컬·메타만 저장** — 대화 원문 미저장, 127.0.0.1 전용, 네트워크 전송 없음.
5. **비개발자도 더블클릭 실행**(Windows exe) — 설치·터미널 없이 대시보드까지.

## Non-goals (YAGNI)

- **팀/멀티유저 집계** — 1머신 1사용자. 여러 사람 로그를 한 대시보드에 모으지 않는다.
  (`db.users` 테이블은 향후 발판으로 비워둘 뿐 미사용.)
- **신규 도구 전체 파서** — 확장점(파서 인터페이스)만 열어두고, Claude Code·Codex 외는
  커뮤니티/후속 과제.
- **구독 정액제 전용 UI**(rate-limit·토큰량 관점) — 1차 타깃은 종량제. 구독은 추정치 추적만.
- **통화 환산** — 비용은 USD 단일 표기. 환율 변환 없음.
- **클라우드 동기화·계정** — 로컬 도구이므로 인증·서버 없음.

## 핵심 기능

| 영역 | 기능 |
|---|---|
| 수집 | Claude Code(`~/.claude`) · Codex(`~/.codex`) 로그 증분 ingest, raw 30일 휘발 대비 archive |
| 예산 | provider별 예산(Claude 월간 / Codex 주간 누적·월÷4). 도입일(`budget_start`) 지정 가능. 웹 `/settings` 또는 config. 0 = 추적 전용 |
| 번다운 | (Claude) 한도 대비 누적 지출·일평균·월말 예상·소진 예상일·on/over. (Codex) 주간 누적 한도(W×N) 대비 이번 주 가용액 |
| 복기 | 프로젝트/세션/차원(모델·스킬·브랜치)별 비용·캐시율·토큰 구성비, 세션 작업요약, 주/월 토글 + 사용자 지정 날짜 구간 + 과거 탐색 |
| 효율 코치 | 캐시 활용 낮음·캐시 재구축·web_search 과다·단가 미식별·월말 초과 예상 경고 카드 |
| 단가 | 공개 API 단가 기본 제공 + `pricing_overrides`로 사용자 단가 덮어쓰기·새 모델 자가 추가. 단가 변경 시 비용 자동 재계산. 단가 커버리지 진단(미식별·오매칭·거친매칭)을 settings·overview·CLI로 노출 |
| 신선도 | 마지막 ingest 경과·가장 오래된 raw 나이 경고(유실 위험 노출) |
| 배포 | Windows onefile exe(pywebview 네이티브 창), 인앱 업데이트 배너 |

## 프라이버시 원칙 (비협상)

- 토큰 **메타데이터만** 저장(토큰 수·시각·프로젝트·모델). 프롬프트·응답 원문 미저장.
- 전 과정 로컬. 웹은 `127.0.0.1`만 바인딩. raw archive도 로컬 전용, 외부 전송 없음.
- README/UI에 프라이버시 경계를 명시한다.

## 성공 기준 (정성)

- 종량제 사용자가 **월 중** 자기 소진 속도를 보고 행동을 바꿀 수 있다(소진 예상일 제공).
- 비개발자가 exe 더블클릭만으로 대시보드까지 도달한다.
- 조직 고유 정보(조직 예산 등급 정책·개인 식별자) 없이 전 세계 사용자가 그대로 쓸 수 있다.
